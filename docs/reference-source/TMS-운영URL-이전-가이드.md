3. ten에서 **「팀 공유본 가져오기」** → 이후 **본인 탭 → 팀 공유 저장**

## 2. 일지 — localStorage vs Blob

| 저장 위치 | 범위 | 용도 |
|-----------|------|------|
| **브라우저 localStorage** | **URL + 브라우저**마다 분리 | 작성·저장의 1차 본 |
| **클라우드 (Blob)** | **ten**에서만 | 「팀 공유 저장」으로 올린 **최신 공유본** |
| **옛 URL Blob** | suspend | 쓰기·가져오기 **불가** — JSON만 신뢰 |

**자주 헷갈리는 점**

- 팀장 ten 화면에 B/C가 보여도 → **팀장 브라우저 localStorage**일 뿐, B/C PC에 **자동 전파되지 않음**.
- Chrome에 최신, Safari 비어 있음 → **정상** (브라우저별 분리).
- Chrome 최신이 Safari에 안 보임 → Chrome에서 **다시 「팀 공유 저장」** 후 다른 브라우저에서 **가져오기**.

## 3. 팀장(A) — 일지 JSON 이전 (필요할 때만)

ten에 이미 최신이면 **§3 생략** → 바로 **「팀 공유 저장」**(§1-A 4단계).

### 3-1. 옛 URL에서 백업 (관리자 화면)

1. https://okestro-edu-team-tms.vercel.app/admin?module=journal  
   (구 URL: `?mode=edit&access=leader&module=journal`)
2. **A · B · C** 탭 확인
3. **「백업용 JSON 다운로드」** → `journal-snapshot-….json`

> 한 파일에 **A/B/C 슬라이스**가 모두 들어갈 수 있습니다 (`memberJournals`).  
> 단, **B/C가 본인 PC에서만 쓴 내용**은 팀장 PC 옛 URL에 **없을 수 있습니다** (§4).

### 3-2. ten으로 가져오기 (관리자만)

1. https://edu-team-tms-ten.vercel.app/admin?module=journal  
2. **「백업 가져오기」** → JSON 선택 (기존 ten 내용과 **병합**, 전체 삭제 아님)
3. **A 탭**(필요 시 B/C 탭) **「팀 공유 저장」**

### 3-3. 이후 습관

| 할 일 | 주기 |
|--------|------|
| 일지 작성 | 매일 (**ten만**) |
| **팀 공유 저장** | 팀에 맞출 때·하루 마감 시 (**필요할 때만**) |
| **백업용 JSON 다운로드** | 주 1회 **비상 백업** 권장 |

## 4. 팀원 (B · C) — 본인 일지 이전

구성원 일지는 **본인 맥북·옛 URL 브라우저** localStorage에만 있는 경우가 많습니다.

### UI에서 알아둘 것 (ten)

| 버튼 | `/admin` (관리자) | `/wschoi` · `/hyshin` (B/C) |
|------|:-----------------:|:---------------------------:|
| **백업용 JSON 다운로드** | ✅ | ❌ |
| **백업 가져오기** (전체 병합) | ✅ | ❌ |
| **조회용 JSON 가져오기** | — | ✅ (타인 일지만, 본인 유지) |
| **팀 공유본 가져오기** | ✅ | ✅ |
| **팀 공유 저장** | ✅ (탭별) | ✅ (본인 탭) |

→ ten **「백업 가져오기」** 는 **관리자가** 합니다.

### 방법 A — 팀장이 병합 (B/C 데이터 이전 시 권장)

1. B/C가 **본인 기기·옛 URL**에서 일지 확인 (JSON 확보 방법은 팀 내부 협의 — 관리자 화면에 다운로드 버튼 없음)
2. JSON 파일을 팀장에게 전달 (슬랙·메일)
3. 팀장이 ten **`/admin` 일지**에서 **「백업 가져오기」** → **팀 공유 저장**
4. B/C는 ten 북마크 → **「팀 공유본 가져오기」** → 이후 본인 **팀 공유 저장**

### 방법 B — 팀 공유만 (가벼운 전환)

1. B/C 북마크를 ten으로 변경
2. **「팀 공유본 가져오기」**로 팀장이 올린 A(·타인) 일지 조회
3. 본인 탭은 ten에서 **새로 작성** → **팀 공유 저장**
4. 옛 일지는 옛 URL JSON만 보관해도 됨

### 방법 C — 아직 전환 안 함

B/C는 **옛 URL만** 써도 되나, **전환 시점**에 §4 방법 A 또는 B를 한 번 수행하세요.

## 7. 팀 공유 저장 · Blob 용량

「팀 공유 저장」을 여러 번 눌러도 **파일이 쌓이지 않습니다.** 같은 경로를 **덮어씁니다.**

| 종류 | Blob 경로 |
|------|-----------|
| 일지 | `journal/live-latest.json` |
| 장부 | `ledger/live-latest.json` |
| KPI 운영 | `kpi-operational/live-latest.json` |
| 향상 과제 | `improve-projects/live-latest.json` |

- 용량은 **저장 횟수**가 아니라 **일지·장부 내용 크기**에 비례합니다 (팀 규모면 보통 KB~수 MB 수준).
- **분기별 Blob 삭제 루틴은 불필요**합니다.
- 대신 **주 1회 JSON 백업 다운로드**(비상 복구)를 권장합니다.

## 8. Blob 반영 확인 (다른 브라우저)

Chrome에서 **「팀 공유 저장」** 후, Safari 등 **다른 브라우저**로 검증할 때:

| 화면 | A 본인 일지 Blob 검증 |
|------|------------------------|
| **`/admin?module=journal` A 탭** | ✅ **권장** — 전체 공유본 병합 가져오기 |
| **`/yhkim`** | ⚠️ 본인 슬라이스는 **로컬 유지** 우선 — 빈 Safari에서 A Blob 검증용으로 부적합 |

**절차 (시크릿 창 권장)**

1. Chrome `/yhkim` — 최신 확인 → **「팀 공유 저장」** (성공 토스트)
2. Safari 시크릿 → `/admin?module=journal&year=2026&month=6` — A 탭
3. **「팀 공유본 가져오기」** → `팀 공유본을 병합했습니다 (클라우드, …)` 확인
4. Chrome과 같은 날짜·업무가 보이면 OK

**토스트**

- `가져온 공유본과 이 브라우저 내용이 같습니다` → 실패 아님 (이미 동기화됨)
- Chrome만 더 최신 → Chrome에서 **다시 팀 공유 저장** 후 반복

## 10. FAQ

### Q. 옛 URL 「팀 공유본 가져오기」로 ten에 못 가져오나요?

**A.** 옛 사이트 Blob은 **suspend**입니다. **JSON 백업**이 가장 확실합니다.

### Q. 같은 맥인데 Safari만 비어 있어요.

**A.** 정상입니다. **브라우저마다 localStorage가 분리**됩니다. §8대로 **팀 공유 저장 → 가져오기**로 확인하세요.

### Q. 「팀 공유 저장」을 자주 하면 Blob 용량이 쌓이나요?

**A.** 아닙니다. `live-latest.json` **덮어쓰기**입니다. §7 참고.

### Q. ten에 A 데이터 있는데 §3 JSON 이전을 또 해야 하나요?

**A.** **아니요.** 최신이면 **팀 공유 저장**만 하면 됩니다 (§1-A).

### Q. B/C 「백업 가져오기」는 본인이 하나요?

**A.** ten UI상 **관리자 `/admin`만** 전체 **백업 가져오기**가 있습니다. B/C는 §4 방법 A·B를 따르세요.

### Q. 카카오 API(점심 메뉴)는?

**A.** ten Vercel 프로젝트에 `KAKAO_REST_API_KEY` 설정이 필요할 수 있습니다. 일지·장부 이전과 별도입니다.
