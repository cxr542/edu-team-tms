# Blob 중단 기간 — 장부·일지 운영 가이드

> **⚠️ 역사 문서 (옛 계정 `okestro-edu-team-tms` 전용)**  
> **현재 운영:** https://edu-team-tms-ten.vercel.app — Blob 정상. 이 페이지는 **2026-06 이전 Blob suspend** 당시 절차 보관용입니다.  
> 새 환경: [접속 URL · 북마크](./TMS-접속URL-북마크.md), [운영 URL 이전 가이드](./TMS-운영URL-이전-가이드.md)

> **상황 (당시):** Vercel Blob 저장소가 **suspend(일시 중단)** 된 동안, TMS의 **클라우드 공유 저장·조회 즉시 반영**이 동작하지 않습니다.  
> **당시 운영 사이트:** https://okestro-edu-team-tms.vercel.app  
> **마지막 갱신:** 2026-06-15

이 문서는 Blob이 **복구되기 전까지** 팀장(A)과 팀원(B·C)이 **장부**와 **일일 업무일지**를 어떻게 운영해야 하는지 정리합니다.

---

## 1. 먼저 알아둘 것

| 구분 | Blob 정상 | **지금 (Blob suspend)** |
|------|-----------|-------------------------|
| 브라우저 **로컬 저장** (작성·수정) | ✅ | ✅ **계속 가능** |
| **팀 공유 저장** / **지금 조회에 반영** (POST) | ✅ | ❌ **실패** (`store has been suspended`) |
| **조회 GET** (장부·일지 API) | 최신 Blob | ⚠️ **배포 시점 백업 JSON** (`public/*-snapshot.json`) |
| 자동 동기화 | 꺼짐(수동 정책) | 꺼짐 |

- **localhost**와 **운영 URL**의 데이터는 **서로 다릅니다.** 운영 북마크 URL에서만 팀 공유를 하세요.
- 역할별 URL은 [TMS 접속 URL · 북마크](./TMS-접속URL-북마크.md)를 따르세요.

## 2. 한눈에 보기 — 누가 무엇을 하나요?

### 장부 (팀 빌딩비)

| 역할 | 해야 할 일 (Blob 중단 중) | 주기 |
|------|---------------------------|------|
| **팀장 A** | 지출 **입력·수정** (팀장 URL) | 수시 |
| **팀장 A** | 조회 반영: **JSON 백업 → 배포** (아래 §3) | **지출 반영할 때마다** |
| **팀원 B·C** | **장부 조회 URL**로 잔액·내역 확인 | 필요 시 |
| **팀원 B·C** | 장부 **편집·조회 반영 없음** | — |

### 일일 업무일지

| 역할 | 해야 할 일 (Blob 중단 중) | 주기 |
|------|---------------------------|------|
| **팀장 A** | 본인(A) 일지 작성 | **매일** |
| **팀장 A** | B·C 탭에서 확인·코칭·대리 입력 | 필요 시 |
| **팀장 A** | B·C에게 받은 **백업 JSON 병합** (`백업 가져오기`) | **주 1회 이상** 또는 승인 전 |
| **팀원 B·C** | **본인 일지 URL**에서 작성 | **매일** |
| **팀원 B·C** | **백업용 JSON 다운로드** → 팀장에게 전달 | **주 1회** 또는 팀장 요청 시 |
| **팀원 B·C** | KPI2 향상 과제: **팀장 JSON 가져오기** | 팀장 배포 시 |

## 3. 장부 — 팀장 (A · 총무)

### 3-1. 일상 업무

1. **팀장 URL** 접속: `?mode=edit&access=leader`  
2. 지출 **추가·수정·삭제** — 변경은 **이 브라우저에 즉시 저장**됩니다.
3. **「지금 조회에 반영」** 은 Blob 중단 중 **실패할 수 있습니다.** 실패해도 로컬 작성본은 유지됩니다.

### 3-2. 팀원이 볼 수 있게 반영하기 (필수)

팀원 장부 조회는 **서버 스냅샷(배포 JSON)** 기준입니다. 아래 **A 또는 B** 중 가능한 방법을 씁니다.

#### 방법 A — JSON 백업 + 배포 (Blob 없이 **권장**)

1. 팀장 화면 **장부**에서 **「장부 JSON 백업 다운로드」** 클릭 → `ledger-snapshot.json` 저장  
2. 저장소 담당자(또는 팀장 PC)에서:

```bash
cd edu-team-tms
npm run publish:team -- ~/Downloads/ledger-snapshot.json
npm run deploy:vercel
```

3. 배포 완료 후 팀원 **장부 조회 URL**에서 **새로고침**으로 확인

#### 방법 B — 「지금 조회에 반영」 (Blob 복구 **이후**에만)

Blob이 풀리면 버튼 한 번으로 조회 URL에 올릴 수 있습니다. **지금은 실패가 정상**입니다.

### 3-3. 팀장 체크리스트

- [ ] 지출 입력 후 **JSON 백업** 또는 배포 반영 요청
- [ ] 팀원에게 “장부 갱신됐다”고 **배포 후** 알림
- [ ] 조회 화면이 **오래된 날짜**면 배포가 안 된 것 — §6 FAQ 참고

---

## 4. 장부 — 팀원 (B · C)

### 4-1. 할 일

1. **본인 장부 조회 URL** 북마크 (예: B)  
   `?mode=view&module=ledger&member=B&year=2026&month=6`
2. 잔액·거래 내역 **조회만** — 편집 불가
3. **「새로고침」** 으로 최신 반영 확인 (자동 갱신 없음)

### 4-2. 하지 않을 일

- 장부 **입력·수정** 시도 (권한 없음)
- 「지금 조회에 반영」 (팀장 전용)

### 4-3. 이상할 때

- 조회 내용이 **며칠~몇 주 전**이면 → 팀장에게 **장부 JSON 배포** 요청
- URL에 `year`·`month`가 맞는지 확인

## 5. 일일 업무일지 — 팀장 (A)

### 5-1. 일상 업무

1. **일지 URL:** `?mode=edit&access=leader&module=journal`
2. **A 탭**에서 본인 업무 작성 (매일)
3. **B·C 탭**으로 전환해 팀원 일지 확인·코칭

### 5-2. Blob 중단 중 공유 방법

| 버튼 | Blob 중단 중 |
|------|----------------|
| **팀 공유 저장** / **팀 공유본 가져오기** | ❌ 실패 — **누르지 않아도 됨** |
| **백업용 JSON 다운로드** | ✅ 일지 + **KPI 승인 상태**(`kpiApproval`) 포함 |
| **백업 가져오기** | ✅ 팀원 JSON 병합 (팀장·A만) — 일지·승인 요청 상태 함께 병합 |

**팀원 일지 수집 절차 (권장):**

1. B·C에게 **매주 금요일**(또는 승인 직전) **「백업용 JSON 다운로드」** 파일을 Slack/메일로 요청
2. 팀장 브라우저에서 **「백업 가져오기」** 로 B·C 파일 **순서대로** 병합
3. 팀 KPI **승인·월 확정** 전에 B·C 탭 내용 확인
4. (선택) 병합 후 **「백업용 JSON 다운로드」** → B·C에게 전달 — 팀원은 **「조회용 JSON 가져오기」**로 타인 일지 조회

### 5-3. KPI2 향상 과제 (운영 목록)

Blob 공유 UI는 꺼져 있습니다. **JSON 파일**만 사용합니다.

1. **팀 KPI 관리** → KPI2 탭 → **「구성원 전달용 JSON 다운로드」**
2. B·C에게 파일 전달
3. B·C는 일지 **향상 과제 패널**에서 **「팀장에게 받은 JSON 가져오기」**

## 6. 일일 업무일지 — 팀원 (B · C)

### 6-1. 일상 업무 (매일)

1. **본인 일지 URL** 북마크 (예: B)  
   `?mode=edit&member=B&module=journal`
2. 당일 업무 **입력·수정** — **이 브라우저에 저장** (본인 탭만)
3. **A · B · C** 탭으로 팀원 일지 **조회** 가능 — 타인 탭은 **조회 전용**(편집·승인 요청 불가)
4. **「팀장 승인 요청」** (KPI1/KPI2 확정·효과 건) — **본인 탭**에서만

> 다른 팀원 탭이 비어 있으면 **「조회용 JSON 가져오기」**(§6-2a)로 팀장이 보낸 백업을 불러오세요.

### 6-2. 팀장에게 보낼 것 (주 1회 권장)

1. 일지 화면 **「백업용 JSON 다운로드」**
2. 파일명 예: `journal-snapshot-2026-06-15T….json` (일지 + KPI1/KPI2 승인 요청 상태 포함)
3. Slack/메일로 **팀장(A)** 에게 전달

### 6-2a. 팀장에게 받을 것 — **조회용 JSON** (타인 일지 보기)

1. 팀장이 B·C 백업을 **병합한 뒤** `백업용 JSON 다운로드`로 만든 파일(또는 팀 전체 일지가 담긴 JSON)을 B·C에게 전달
2. B·C 일지 화면 상단 **「조회용 JSON 가져오기」** → 파일 선택
3. **A · C**(또는 A · B) 탭에서 타인 일지 **조회** — **본인 탭 일지는 덮어쓰지 않음**
4. KPI 승인 상태(`kpiApproval`)는 반영하지 않습니다 (조회 전용)

> 팀장용 **「백업 가져오기」** 와 다릅니다. B·C는 병합·승인 데이터 반영 없이 **타인 일지만** 이 브라우저에 채웁니다.

### 6-3. 팀장에게 받을 것 (향상 과제)

- **향상 과제 JSON** — 일지 하단 패널 **「팀장에게 받은 JSON 가져오기」**
- **팀 공유본 가져오기** 버튼은 B·C 일지에 **표시되지 않습니다** (정상)

### 6-4. 하지 않을 일

- **팀 공유 저장** (버튼이 없거나, 있어도 Blob 중단으로 실패)
- **백업 가져오기** (팀장 전용 — B·C는 **조회용 JSON 가져오기**만 사용)
- 다른 팀원 탭에서 **작성·수정** (조회만 가능 — 본인 `member=` 탭에서 작성)
- 다른 팀원 URL로 작성 (본인 `member=` URL만 사용)

## 7. 자주 묻는 문제

### Q. B/C인데 다른 팀원 일지 탭이 비어 있습니다.

**A.** 일지는 **브라우저별 localStorage**입니다. 팀장이 병합·전달한 JSON을 **「조회용 JSON 가져오기」**로 불러오면 A/C 탭에 표시됩니다. 본인 탭 내용은 그대로 유지됩니다.

### Q. 「팀 공유 저장」을 눌렀는데 실패합니다.

**A.** Blob suspend 때문입니다. **JSON 파일**로 주고받으세요 (§5·§6).

### Q. 장부 조회가 팀장 화면과 다릅니다.

**A.** 팀장은 **로컬 작성본**, 팀원 조회는 **마지막 배포 JSON** 입니다. 팀장이 §3-2 **배포** 후 팀원이 새로고침해야 맞습니다.

### Q. 「팀 공유본 가져오기」 토스트만 뜨고 내용이 안 바뀝니다.

**A.** API가 **옛 백업 JSON**만 돌려주거나, 이미 로컬과 같을 수 있습니다. **팀장이 받은 JSON 파일**으로 `백업 가져오기`를 쓰세요.

### Q. Blob은 언제 풀리나요?

**A.** Vercel Dashboard → Storage → Blob 상태 확인. Hobby는 한도 리셋·**[Vercel 새 계정 연결](../vercel-new-account-setup.md)**·Pro 업그레이드·지원 문의가 필요할 수 있습니다. **풀리기 전까지는 이 문서 방식을 유지**합니다.

### Q. Blob이 복구되면?

- **장부:** 「지금 조회에 반영」 재시도 가능 (여전히 **수동** 권장)
- **일지:** 팀장 화면 **팀 공유 저장·가져오기** 재확인 (정책상 자동 동기화는 사용 안 함)
- 이 문서의 **JSON 백업**은 그대로 비상용으로 유지 가능
