# 2026 교육팀 핵심 역량 평가 — 설계 노트

엑셀 초안(`팀역량레벨평가_초안_*.xlsx`)과 생성 스크립트(`build_level_workbook.py`)를 바꾸기 **전에** 팀장·운영자가 합의할 내용을 정리한 문서입니다. 수치·문구는 본부 지침에 맞춰 조정 가능합니다.

---

## 1. 목적과 원칙

| 항목 | 내용 |
| --- | --- |
| 목적 | 본부 **5단계 레벨**과 정합되는 팀 내 역량 평가·성장 대화의 공통 언어 확보 |
| 척도 | 5차원 루브릭 충족 여부를 **0.2 단위**로 누적해 정수 레벨에 가산하는 방식 |
| 역할 | 팀원 **자체평가**와 팀장 **최종 판단**을 분리하되, 월간·분기 집계에는 **본부 가중치(40:60)**를 반영 |
| 운영 | 차원별 미충족은 **코칭·분기 과제**와 연결하고, 필요 시 **캡**으로 “한 단계 올라가기”의 조건을 명시 |

---

## 2. 본부 정합성

### 2.1 최종 역량 점수(월간 Raw의 KPI 연계용)

- **제안 산식:**  
  `최종 역량(종합)` = `(자체평가 종합) × 0.4` + `(팀장평가 종합) × 0.6`
- **해석:** 자체평가는 성찰·입력 정확도에 가깝고, 팀장 평가는 조직 기준 반영 비중이 크다는 본부 **40:60** 지침과 맞춤.
- **엑셀 반영 시:** `월간평가_RAW`에 자체·팀장 종합을 각각 두 열로 두고, 집계·분기 평균이 참조하는 **대표 열**을 “최종”으로 고정하는지(권장) 여부를 팀에서 확정.

### 2.2 허용 소수

- 종합·가중 결과는 **0.2 단위**로 표시·입력 제한(MROUND 등)을 유지할지, 가중 직후만 반올림할지 결정.

## 3. 직군(역할) 정의

목록 값은 엑셀 데이터 유효성 검사와 동일하게 쓰는 것이 좋습니다.

| 코드(예시) | 대상 |
| --- | --- |
| `전문강사` | 강의·교안·기술 전달 중심 |
| `기획/운영` | 커리큘럼·운영·이해관계자 조율·DX·자동화 기획 중심 |

※ 표기 통일: `기획운영` / `기획·운영` 등 오타 방지를 위해 **한 가지**로 고정.

---

## 4. 루브릭 시트 분리(방향 2)

### 4.1 공통 축

- **차원(5개):** 자율성, 범위·난이도, 협업·영향, 품질·완결성, 전문성·표준화  
- **열:** Level 1~5 관찰 문구(행 순서는 “소수 누적 시트”의 차원 순서와 맞출 것)

### 4.2 시트 이름(안)

| 시트명 | 보완 방향(관찰 지표 힌트) |
| --- | --- |
| `루브릭_강사` | 신기술(K8s/Ceph 등) **자산화**, 강의 만족도, **교안 정밀도**·표준화 |
| `루브릭_기획운영` | **이해관계자 조율**, 운영 프로세스 **자동화**, **DX 기획**·로드맵 |

기존 단일 `루브릭` 시트는 위 둘로 **대체**하는 것을 전제로 합니다.

---

## 5. 소수 누적 우선순위(직군별) — 합의 필요

차원 열 자체(E~I 등)는 **동일**하게 두고, “어느 차원부터 순차적으로 +0.2를 줄지”만 직군별로 바꾸는 방식이 구현·설명 모두 단순합니다.

| 직군 | 제안 누적 순서(선행 충족 → 다음 +0.2) | 의도 |
| --- | --- | --- |
| (기본·미선택 시) | 자율성 → 범위·난이도 → 협업·영향 → 품질·완결성 → 전문성·표준화 | 현행 초안과 동일 |
| 전문강사 | 전문성·표준화 → 품질·완결성 → 협업·영향 → 범위·난이도 → 자율성 | “깊이·표준”을 앞에 |
| 기획/운영 | 협업·영향 → 자율성 → 범위·난이도 → 품질·완결성 → 전문성·표준화 | “조율·주도”를 앞에 |

팀장님이 **순서 자체**를 바꾸길 원하시면 위 표만 수정한 뒤 엑셀·스크립트에 반영하면 됩니다.

---

## 6. 캡(Cap) 규칙 — 합의 필요

| 직군 | 제안 캡 기준 |
| --- | --- |
| 기본 | 품질·완결성 미충족 시 누적 상한(예: 0.4) 등 **현행과 동일** 유지 |
| 전문강사 | 품질·완결성 **및** 전문성·표준화가 모두 충족일 때만 캡 해제(둘 중 하나라도 미충족이면 상한 적용) |
| 기획/운영 | 협업·영향 미충족 시 상한 적용 |

구체 수치(0.4, 1.0 등)는 본부 예시와 팀 내례에 맞춰 조정.

---

## 7. 시트·수식 매핑(구현 체크리스트)

엑셀/스크립트 작업 시 아래 순서를 권장합니다.

1. **직군 열 추가**  
   `소수_누적_자체평가` / `소수_누적_팀장`에 동일 키(평가월|이름)로 직군을 맞출 것. 팀장 시트는 기본값으로 자체평가 직군을 가져오고, 예외만 덮어쓰기.
2. **누적·캡 수식**을 직군별 분기로 교체(위 §5, §6).
3. **`루브릭_강사` / `루브릭_기획운영`** 작성 후 기존 `루브릭` 제거.
4. **`월간평가_RAW`**  
   - 자체 종합, 팀장 종합 열 분리  
   - `0.4*자체 + 0.6*팀장` 최종 열  
   - **`분기집계`가 평균 내는 열**을 최종 열로 고정할지 명시(권장: 최종 열 = 분기 평균 입력).
5. **`코드북`**에 직군 값, 누적 순서, 캡, 40:60, 시트 이름 변경을 문장으로 반영.

---

## 8. 운영 메모(성장 가이드)

- 차원별 **미충족**은 다음 분기 **M/M 과제**나 교안·운영 과제와 1:1로 연결해 기록하면 “점수”가 아니라 **성장 가이드**로 남습니다.
- 레벨 **캡**은 팀 전체 기술/품질 하한을 올리는 장치로, 도입 시 팀원에게 사전 공지하는 것이 좋습니다.
