# 일지 ↔ TMS KPI 연계 가이드

일지 화면 상단 **「이번 달 예상」** 만 보여 주고, 아래 표·설명은 이 문서에서 확인합니다.

**공식 기록:** TMS (`module=kpi`) · **엑셀**은 분석·백업 export 시 스냅샷만.

**일지 연동 구성원:** A (김윤형) — B/C는 팀 KPI **월 확정**에서 M/M 직접 입력.

## 이번 달 예상 (일지 상단)

| 항목 | 의미 |
|------|------|
| **N주** | 해당 월 **주간 M/M 합계** 행 수 (엑셀 `01c_KPI1_주간메모`) |
| **효과 M건** | `kpi2Effect` 켠 **완료** 업무 수 |

실시간 집계·제출·승인은 [팀 KPI 관리](http://localhost:3000/?mode=edit&module=kpi)에서 합니다.

## 일지 입력 → KPI 반영

| 일지에서 하는 일 | TMS / KPI | 엑셀 시트 (분석용) |
|------------------|-----------|-------------------|
| 일별 업무 (실작업 h, M/M 축) | **업무 리소스 가동률** — 업무MM · 생산향상MM | `01c_KPI1_주간메모` |
| KPI 탭 **주간 메모** (일지 「금주」와 별도) | 가동률 — 주간메모 열 | `01c_KPI1_주간메모` |
| **생산성 효과** (기준h·실작업·향상과제) | **업무 리소스 생산성** — 효과 건만 | `02_KPI2_입력` |
| 휴일 M/M · 휴일 메모 | 가동률 — 휴일MM (생산성 효과 제외) | `01c_KPI1_주간메모` |

### M/M · 생산성 규칙 (요약)

- **가동률:** **완료** 체크한 업무의 실작업 ÷ 8 → M/M. AI 카테고리·수동 「향상」→ **생산향상MM**.
- **생산성:** `kpi2Effect.enabled` 이고 **완료**된 행만 실작업 집계. 승인 큐에는 「승인 요청」을 누른 건만. 생산성% = Σ기준시간 ÷ Σ실작업 (승인 건 집계).
- **백업 JSON:** 구성원별 `kpiApproval`(월확정·KPI2 승인 상태)이 포함됩니다. 팀장 `백업 가져오기` 시 병합됩니다.
- **팀 공유:** A/B/C 모두 운영 URL에서 **「팀 공유 저장」** / **「팀 공유본 가져오기」** (수동). B/C 가져오기는 **본인 일지 유지·타인만 갱신**. **향상 과제**도 KPI2 팀 공유 Blob 사용(JSON 예비는 Blob 꺼짐 시만).
- **개발·개선 투자** (향상 M/M만, 효과 OFF) → 가동률만, 생산성 리스트에는 **안 나옴**.
