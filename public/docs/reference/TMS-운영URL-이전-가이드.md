# TMS 운영 URL 이전 가이드 (2026-06)

> **새 운영 URL:** https://edu-team-tms-ten.vercel.app  
> **이전 URL (사용 중단):** https://okestro-edu-team-tms.vercel.app  
> **이유:** 이전 Vercel 계정 Blob 한도 초과 → 새 계정에서 클라우드 공유 재개

---

## 1. 팀 전체 — 단계별 이전

### 1-A. 파일럿 (지금 — 팀장만)

1. **팀장(A)** 만 [접속 URL · 북마크](./TMS-접속URL-북마크.md) §0·§2 **새 URL** 사용
2. 옛 URL에서 **백업 JSON** → 새 URL **백업 가져오기** (일지 §3, 장부 §6)
3. **일지 A/B/C · 향상 과제 · 장부** 각각 **팀 공유 저장** / **지금 조회에 반영** (Blob 시드)
4. 안정화 확인 후 구성원에게 §1-B 안내

### 1-B. 구성원 전환 (팀장 안내 후)

1. B/C **북마크를 새 URL**로 교체 (§4·§5)
2. 본인 옛 URL 일지가 있으면 **백업 가져오기** (§4 방법 A)
3. 팀장이 올린 공유본이 있으면 **팀 공유본 가져오기**로 맞추기
4. 이후 **본인 탭 → 팀 공유 저장** 습관화

> **옛 URL에서는 더 이상 작성하지 마세요** (팀장 파일럿 중에도 구성원이 옛 URL에 쓰면 나중에 이중 관리됩니다).

---

## 2. 일일 업무일지 — 어디에 저장돼 있나?

| 저장 위치 | 설명 |
|-----------|------|
| **브라우저 localStorage** | 그 **URL + 그 브라우저**에만 있음. 옛 URL과 새 URL은 **완전히 별개** |
| **클라우드 (Blob)** | **새 URL**에서만 동작. 「팀 공유 저장」으로 올림 |
| **옛 URL Blob** | 한도 초과로 **쓰기 중단** — 최신 일지가 클라우드에 없을 수 있음 |

→ **옛 URL에서 쓴 내용은, 그 브라우저에서 JSON으로 꺼내와야** 새 URL로 옮길 수 있습니다.

---

## 3. 팀장(A) — 일지 통합 이전 (권장)

팀장 PC에서 옛 URL에 A/B/C 일지가 가장 많이 모여 있을 때:

### 3-1. 옛 URL에서 백업

1. https://okestro-edu-team-tms.vercel.app/?mode=edit&access=leader&module=journal  
2. (선택) **A · B · C** 탭을 각각 열어 화면이 최신인지 확인  
3. **「백업용 JSON 다운로드」** → `journal-snapshot-….json` 저장

> 한 파일에 **A/B/C 구성원 슬라이스**가 모두 들어 있습니다 (`memberJournals`).

### 3-2. 새 URL로 가져오기

1. https://edu-team-tms-ten.vercel.app/?mode=edit&access=leader&module=journal  
2. **「백업 가져오기」** → 방금 JSON 선택  
3. 내용 확인 후, **A · B · C** 탭마다 **「팀 공유 저장」** (클라우드에 올림)

### 3-3. 이후 운영

| 할 일 | 주기 |
|--------|------|
| 본인·팀원 일지 작성 | 매일 (새 URL) |
| **팀 공유 저장** | 팀원에게 공유할 때·하루 마감 시 (필요할 때만) |
| **백업용 JSON 다운로드** | 주 1회 비상 백업 권장 |

---

## 4. 팀원 (B · C) — 본인 일지 이전

구성원이 **본인 맥북·브라우저**에서 옛 URL로만 썼다면, **그 기기에서만** 일지가 있습니다.

### 방법 A — 본인이 직접 (권장)

1. **옛 URL** + **본인 북마크** (`?mode=edit&member=B&module=journal` 등)  
2. **「백업용 JSON 다운로드」**  
3. **새 URL** 북마크로 접속 → **「백업 가져오기」**  
4. 이후 작성·북마크는 **새 URL만** 사용 → 마감 시 **「팀 공유 저장」**

### 방법 B — 팀장에게 파일 전달 (예비)

1. 본인 기기에서 JSON 다운로드 → 팀장에게 전달 (메일·슬랙)  
2. 팀장이 새 URL에서 **백업 가져오기**로 병합 후 **팀 공유 저장**  
3. 팀원은 새 URL에서 **「팀 공유본 가져오기」**

### 방법 C — 팀 공유만으로 맞추기

팀장·다른 팀원이 이미 **팀 공유 저장**을 했다면, 새 URL에서 **「팀 공유본 가져오기」**로 타인 일지를 볼 수 있습니다.  
본인 탭이 비어 있으면 **방법 A**로 본인 JSON을 먼저 가져온 뒤, 앞으로는 **본인 탭 → 팀 공유 저장**을 사용하세요.

## 5. 역할별 작성 요약 (새 URL)

### 팀장 (A)

| 메뉴 | URL 패턴 | 작성·운영 |
|------|----------|-----------|
| 장부 | `?mode=edit&access=leader` | 지출 입력 → **지금 조회에 반영** |
| 일지 (본인) | `…&module=journal` (A 탭) | 일별 업무 · **팀 공유 저장** |
| B/C 일지 확인 | `…&member=B\|C&module=journal` | 대리 입력 가능 → 해당 탭 **팀 공유 저장** |
| KPI | `…&module=kpi` 등 | 월 확정·승인 |

### 팀원 (B · C)

| 메뉴 | URL 패턴 | 작성·운영 |
|------|----------|-----------|
| **일지** | `?mode=edit&member=B\|C&module=journal` | **본인 탭만** 편집 → **팀장 승인 요청** → **팀 공유 저장** |
| 팀원 일지 조회 | A/B/C 탭 | **팀 공유본 가져오기** (본인 일지 유지, A/C 등 타인만 갱신) |
| 장부 | `?mode=view&member=…&module=ledger` | 조회만 |

자동 저장·자동 동기화는 **없습니다**. 팀 공유는 **수동** 버튼입니다.

---

## 6. 장부 이전

일지와 달리 장부는 **팀장 편집 화면**에서만 JSON으로 옮깁니다.

### 6-1. 옛 URL에서 백업

1. https://okestro-edu-team-tms.vercel.app/?mode=edit&access=leader  
2. **「장부 JSON 백업 다운로드」** → `ledger-snapshot.json` 저장

### 6-2. 새 URL로 가져오기

1. https://edu-team-tms-ten.vercel.app/?mode=edit&access=leader  
2. **「장부 JSON 백업 가져오기」** → 방금 JSON 선택 → 확인  
3. 내용 확인 후 **「지금 조회에 반영」** (팀원 장부 조회용 Blob)

> **「조회 데이터 맞추기」** 는 **서버에 이미 올라간 조회 스냅샷**을 작성 장부에 맞출 때 쓰는 버튼입니다. 옛 URL에서 받은 **로컬 백업 파일**은 **백업 가져오기**를 사용하세요.

### 6-3. 참고

| 상황 | 방법 |
|------|------|
| 팀장 **작성 중** 장부 이전 | 위 6-1 → 6-2 → **지금 조회에 반영** |
| 새 URL Blob만 비어 있고 작성본은 이미 맞음 | **지금 조회에 반영**만 |
| 개발자 수동 배포 (비상) | `npm run publish:team -- ./ledger-snapshot.json` 후 재배포 |

## 7. FAQ

### Q. 옛 URL 「팀 공유본 가져오기」로 새 URL에 못 가져오나요?

**A.** 옛 사이트 Blob은 **쓰기 중단** 상태입니다. 공유본이 **오래된 백업**이거나 비어 있을 수 있습니다. **JSON 백업**이 가장 확실합니다.

### Q. 같은 브라우저인데 새 URL이 비어 있어요.

**A.** 정상입니다. **도메인이 다르면** localStorage가 공유되지 않습니다. JSON 가져오기가 필요합니다.

### Q. 「팀 공유본 가져오기」 후 「내용이 같습니다」만 떠요.

**A.** 실패가 아닙니다. 이미 그 브라우저와 클라우드가 같다는 뜻입니다. **다른 PC·시크릿 창**에서 가져오기를 테스트하세요.

### Q. 카카오 API(점심 메뉴)는?

**A.** 새 Vercel 프로젝트에 `KAKAO_REST_API_KEY` 를 넣어야 합니다. 일지·장부 이전과 별도 작업입니다.
